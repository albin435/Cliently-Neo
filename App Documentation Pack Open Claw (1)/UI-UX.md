# Neo — UI / UX

# Design Philosophy

Neo should feel like:
- a premium internal operating system
- a technical command center
- a clean AI engineering workspace

NOT:
- a generic chatbot
- a futuristic neon AI toy
- a Telegram clone

---

# Main Interface

## Left Sidebar
- new chat
- project workspaces
- execution history
- memory
- timelines
- settings

---

## Main Workspace
A dedicated internal communication interface.

Inspired by:
- Codex
- Cursor
- Discord workspace structure
- Linear

Supports:
- persistent conversations
- execution threads
- task discussions
- live execution updates
- markdown
- code blocks
- execution cards

---

## Right Context Panel
Displays:
- active agents
- current runtime state
- git status
- repository context
- memory references
- execution metadata

---

# Approval Cards

Shows:
- risks
- files affected
- commands
- execution plans
- required tools

Buttons:
- approve
- modify
- reject

---

# UX Principles

- keyboard-first
- low visual noise
- fast interactions
- operational focus
- smooth animations
- structured information hierarchy
