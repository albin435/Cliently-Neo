# Neo — Product Requirements Document

## Overview
Neo is a native macOS AI CTO orchestration platform designed for personal use by Albin.
Neo acts as a supervisory engineering intelligence layer over Antigravity Agents.

Instead of directly managing multiple AI agents, the user interacts only with Neo.

Neo:
- plans
- supervises
- delegates
- reviews
- validates
- synthesizes
- maintains architectural integrity

Neo operates using an approval-based workflow similar to Codex/Cursor style agentic systems.

---

# Core Philosophy

Human → Neo → Agent Swarm

Neo is NOT a chatbot.
Neo is a CTO-grade orchestration framework.

---

# Primary Goals

- Reduce cognitive overload from managing multiple agents
- Maintain architecture consistency
- Preserve project memory and historical decisions
- Supervise agent execution
- Create reliable execution pipelines
- Improve development velocity
- Keep all execution approval-based

---

# Key Features

## Conversational CTO Interface
- Persistent chat interface
- Multi-chat support
- Full chat history
- Searchable conversations
- Model selection per chat

## Supervisor Pipeline
1. Strategic Planning
2. Task Contract Generation
3. Agent Delegation
4. Antigravity Collaboration
5. Review & Validation
6. Executive Summary

## Agent Orchestration
Neo can:
- create sub-agents
- assign technical roles
- monitor execution
- retry failed tasks
- reject low-quality outputs
- supervise architecture compliance

## Approval Gates
Before execution:
- affected files shown
- execution plan shown
- risks shown
- approval required

## Persistent Memory
- SQLite chat memory
- vector memory
- git awareness
- project indexing
- historical decision tracking

## Workspace Awareness
Neo scans:
- repository structure
- git state
- environment files
- Prisma schemas
- configs
- logs
- Antigravity context

## Execution Timeline
Every task has:
- planning logs
- delegation history
- execution states
- review summaries
- replayable history
