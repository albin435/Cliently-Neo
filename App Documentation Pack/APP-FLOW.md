# Neo — App Flow

# Main User Flow

1. User opens Neo
2. Neo restores workspace context
3. User creates/selects chat
4. User gives command
5. Neo analyzes request
6. Neo generates task contract
7. User approves
8. Neo delegates execution
9. Neo supervises execution
10. Neo reviews outputs
11. Neo delivers executive summary

# Retry Flow

If execution quality is poor:
- Neo rejects output
- generates revised directive
- reassigns task
- revalidates output
